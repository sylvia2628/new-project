# MUSE AI OS MVP 規劃

## 目標

第一版先建立公司內部 AI 工作中心，優先解決五件事：

1. AI 秘書首頁：每日工作摘要
2. 客戶 CRM 管理
3. 案場進度管理
4. 工作 / 待辦事項管理
5. 作品管理，含 AI 文案

LINE 官方帳號、Facebook、Instagram、Google Drive、TimeTree 與財務整合放在第二階段。

## 第一階段：內部管理核心

功能：

- AI 秘書每日主動產生工作摘要
- 客戶 CRM 建檔與追蹤
- 案場流程與進度百分比
- 工作待辦與負責人
- 作品資料與 AI 文案草稿

AI 第一版只產生建議與草稿，不自動對外發送訊息。

## 第二階段：外部平台串接

功能：

- LINE Official Account API
- Google Drive API
- Google Calendar API
- TimeTree 同步可行性研究
- Facebook / Instagram API
- Gmail API
- 官網串接
- 財務與毛利分析

第一版不需要提供帳號密碼。未來串接使用官方 API 授權。

## 核心資料庫

第一版至少需要建立：

- 客戶資料庫
- 案場資料庫
- 工作資料庫
- 作品資料庫
- 知識庫
- 使用者與權限資料

## 重要原則

- AI 主動提醒，而不是只被動回答
- 所有客戶回覆需管理者確認後才發送
- 所有社群與官網內容需管理者確認後才發布
- 財務與合約資料需權限控管
- 外部平台只作為入口，資料統一回到 MUSE AI OS 管理

## 建議系統架構

```text
LINE / Facebook / Instagram / Google / 官網
  ↓
外部入口
  ↓
MUSE AI OS
  ↓
AI 秘書 / CRM / 案場 / 工作 / 作品 / 財務
  ↓
團隊協作與管理者審核
```

## 第一版優先順序

1. 確認使用者角色與權限
2. 建立客戶 CRM 欄位
3. 建立案場流程與狀態
4. 建立工作待辦欄位
5. 建立作品管理欄位
6. 建立 AI 秘書首頁的每日摘要格式

## 需要你補充的資料

- 公司內部成員名單與角色
- 每個角色可以看哪些資料、改哪些資料
- 目前 TimeTree 行程管理方式
- 目前客戶追蹤是用 Excel、Google Sheet、Notion 還是人工紀錄
- 目前案場進度如何記錄
- 目前工作待辦如何指派
- 目前作品素材放在哪裡
- 是否有不想接的案件類型
- 是否有最低工程預算
