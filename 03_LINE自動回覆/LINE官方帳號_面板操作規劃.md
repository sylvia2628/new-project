# LINE 官方帳號面板操作規劃

## 目標

目前官方 LINE 資訊不足，客戶加入後不容易快速了解：

- 慕舍設計做什麼
- 有哪些作品
- 收費方式
- 設計與工程流程
- 如何預約諮詢
- 要先準備哪些資料
- 地址與聯絡方式

第一階段建議先調整 LINE 官方帳號的「圖文選單 Rich Menu」與「自動回覆關鍵字」，讓客戶不用等待人工回覆，也可以先取得基本資訊。

## 第一階段建議做法

先不要串接 API，不需要提供帳號密碼。

先用 LINE Official Account Manager 後台完成：

1. 圖文選單 Rich Menu
2. 加好友歡迎訊息
3. 關鍵字自動回覆
4. 預約諮詢引導訊息
5. 客戶資料收集格式

等第一版穩定後，再接：

- LINE Messaging API
- Webhook
- MUSE AI OS CRM
- AI 自動整理客戶摘要

## 建議圖文選單：六格版

建議使用 2 排 3 欄，也就是 6 個按鈕。

```text
┌──────────┬──────────┬──────────┐
│ 作品案例 │ 預約諮詢 │ 收費方式 │
├──────────┼──────────┼──────────┤
│ 服務流程 │ 裝修 FAQ │ 地址電話 │
└──────────┴──────────┴──────────┘
```

### 按鈕 1：作品案例

目的：讓客戶快速看到作品，提升信任感。

建議動作：

- 開啟官網作品頁

連結：

```text
https://mus-design.com.tw/portfolio
```

按鈕文字：

```text
作品案例
```

輔助文字：

```text
住宅、商空、老屋翻修案例
```

### 按鈕 2：預約諮詢

目的：引導客戶提供完整資料，減少來回詢問。

建議動作：

- 第一階段：送出文字「我要預約諮詢」
- LINE 自動回覆引導客戶填資料
- 未來：改成開啟表單或 MUSE AI OS 預約頁

按鈕文字：

```text
預約諮詢
```

自動回覆草稿：

```text
您好，歡迎預約慕舍設計諮詢。

為了讓設計師更快了解您的需求，請協助提供以下資料：

1. 姓名：
2. 電話：
3. 案件地點：
4. 房屋類型：預售屋 / 新成屋 / 老屋翻修 / 商業空間
5. 坪數：
6. 預算區間：
7. 預計裝修時間：
8. 是否有平面圖或現況照片：
9. 喜歡的風格或參考圖片：

收到後，我們會由專人協助評估與安排諮詢。
```

### 按鈕 3：收費方式

目的：降低重複詢價，提高客戶品質。

建議動作：

- 送出文字「收費方式」
- 自動回覆設計費、丈量費、工程費與監工費說明

按鈕文字：

```text
收費方式
```

```text

工程費用另計，會依設計圖定稿版本估價。

舊屋翻修：總工程價 8%

實際費用仍需依現況、坪數、屋況、需求與施工內容討論。
```

## 關鍵字自動回覆建議

| 優惠、折扣、LINE 優惠 | 回覆本月 LINE 諮詢優惠說明 |

## 客戶操作路徑

### 路徑 A：想先看作品

1. 客戶加入 LINE
2. 點「作品案例」
3. 開啟官網作品頁
4. 回到 LINE 問喜歡的風格
5. AI 或人工引導預約諮詢

### 路徑 B：想知道費用

1. 客戶點「收費方式」
2. 自動回覆設計費、丈量費與工程費說明
3. 引導提供地點、坪數、屋況、預算
4. 資料完整後交給人工評估

### 路徑 C：想直接預約

1. 客戶點「預約諮詢」
2. 自動回覆資料填寫格式
3. 客戶提供資料、平面圖或照片
4. 後台建立 CRM 潛在客戶
5. 人工接手安排諮詢

## 圖文選單設計方向

建議風格：

- 色系：米白、深灰、暖棕、低飽和金色
- 文字：簡短清楚，每格 4 字以內為主
- 視覺：以慕舍作品照片、品牌 logo、簡潔線條 icon 為主
- 避免：過多小字、優惠字樣過大、一次塞太多資訊

建議尺寸：

- 第一版可用 LINE 後台提供的圖文選單模板
- 若使用 API 或自行設計圖，需符合 LINE Rich Menu 圖片規格

## LINE 官方規格提醒

依 LINE Developers 官方文件：

- Rich Menu 是顯示在 LINE 官方帳號聊天室底部的選單。
- Rich Menu 由選單圖片、可點擊區域與聊天列組成。
- Rich Menu 不會顯示在 LINE 電腦版。
- 可用 LINE Official Account Manager 或 Messaging API 建立。
- LINE Official Account Manager 適合快速建立、圖形化操作、設定顯示期間與查看點擊統計。
- Messaging API 適合進階客製，例如每個使用者不同選單、postback、日期選擇、分頁切換。
- 同一個 Rich Menu 不能同時用後台與 API 編輯。
- 圖片格式需為 JPEG 或 PNG，寬度 800 到 2500 px，高度至少 250 px，寬高比至少 1.45，檔案最大 1 MB。
- 已上傳到 Rich Menu 的圖片不能直接取代，若要更新圖片，需建立新的 Rich Menu。

參考來源：

- LINE Developers Rich menus overview：https://developers.line.biz/en/docs/messaging-api/rich-menus-overview/
- LINE Developers Use rich menus：https://developers.line.biz/en/docs/messaging-api/using-rich-menus/
- LINE Developers Messaging API reference：https://developers.line.biz/en/reference/messaging-api/

## LINE 後台設定步驟

以下以 LINE Official Account Manager 為方向，實際名稱可能會依 LINE 後台版本略有不同。

1. 登入 LINE Official Account Manager
2. 選擇慕舍官方帳號
3. 找到「圖文選單」或「Rich Menu」
4. 新增圖文選單
5. 選擇 6 格版型
6. 上傳設計好的圖文選單圖片
7. 設定每一格的動作
8. 設定顯示期間
9. 設定聊天列文字，例如「點選服務選單」
10. 儲存並發布
11. 用手機加入官方 LINE 測試每一格是否正常

## 上線前需要確認的資料

- 目前官方 LINE 後台是否已有圖文選單
- 官方 LINE 是否已有加好友歡迎訊息
- 是否已有關鍵字自動回覆
- 本月 LINE 優惠的正確內容
- 是否要開放客戶直接預約到公司諮詢
- 是否有 Google 表單或預約表單
- 是否要讓客戶直接上傳平面圖與現況照片
- 是否要把「台南、屏東」列為可服務地區，或只寫高雄為主

## 第二階段：串接 MUSE AI OS

未來串接後可以做到：

1. 客戶點「預約諮詢」
2. LINE 收集姓名、電話、地點、坪數、預算、風格、照片
3. Webhook 傳到 MUSE AI OS
4. AI 自動建立 CRM 潛在客戶
5. AI 產生客戶摘要與追蹤提醒
6. 管理者確認後再回覆客戶

原則：

- 不提供 LINE 帳號密碼
- 使用官方 API 授權
- AI 不自動發送正式訊息
- 重要回覆需由管理者確認
